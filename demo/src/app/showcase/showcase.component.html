Playground

The toolbar is the engine's native control, auto-built from the phenomenon registry with built-in SVG icons. Pick a phenomenon, then edit everything on the map. For the jet stream: draw freehand, drag the orange break points, click one for the speed dial + vertical FL gauge, double-click to add a break point, drag one off the line to delete it; the blue end points are fixed at 80 kt and reshape the curve.

Per-phenomenon config — optional

The chrome you tune on the right — selection, handles, sliders, the speed dial, the tooltip — is global. Everything specific to one phenomenon lives under phenomena: { <type>: { speed?, flightLevel?, style? } }: its numeric bounds (jet speed, turbulence flightLevel) and its own visual tokens — the shape colour (jet axis + arrow + feathers; turbulence dashed edge + fill), the FL text, the severity glyph. Every phenomenon is one call

The jet stream is the flagship — a smooth directional curve whose barbs, change bars and FL labels are all derived from the break-point metadata you edit on the map. CB (scalloped area) and turbulence (dashed area) are also supported. Each button calls sigwx.draw("…") on the live map; listen for change to get the GeoJSON.

Phenomenon-driven rendering

The phenomenon registry + the per-feature metadata drive the geometry — you edit data on the map and the depiction follows. Jet: wind barbs scale with speed, a change bar replaces feathers at a ±20 kt monotonic step, FL labels at the max-wind & FL-change points, and a vertical extent band once speed ≥ 120 kt. CB: a scalloped edge. Turbulence: a dashed edge.

Change the style at runtime

Less common, but you can restyle after grafting — only what you pass is overridden, the rest keeps its current value.
